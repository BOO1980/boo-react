ES6 Overview

  • ES6 stands for ECMAScript6
  • ECMAScript was created to standardize JavaScript, and ES6 is the 6th version of ECMAScript, it was published in 2015
  • React uses ES6, you should be familiar with some of the features like
    • Classes
    • Arrow Functions
    • Variables

Classes

  • A class is a type of function
  • The properties of the class are assigned inside a constructor method
  • The constructor function is called automatically when the object is initialised

Methods

  • You can create your own methods within the classes
  • You call the method by referring to the object's method name followed by parentheses (parameters would go inside the parentheses)

Class Inheritance

  • To create a class inheritance, use the extends keywords
  • A class created with class inheritance inherits all the methods from another class
  • The super method refers to the parent class

Arrow functions

  • Arrow functions allow us to write shorter function syntax

this

  • The handling of this is also different in arrow function compared to regular functions
  • The arrow functions have no binding to this
  • In regular functions the this keyword represented the object that called the function, which could be the window , the document, a button or whatever
  • with arrow functions, the this keyword always represents the object that defined the arrow function

Variables

  • Before ES6 there were only way of defining your variables: with the var keyword. If you did not define them, they would be assigned to the global object
  • With ES6 there are 3 ways to define variables
    var
    • If you use var outside of a function it belongs in the global scope
    • If you use var inside of a function, it belongs to that function
    • If you use var inside of a block, i.e a for loop the variable is still available outside of that block
    • var has a function scope, not a block scope
    let
    • let is the block scoped version of var and is limited to the block (or expression) where it is defined
    • If you use let inside of a block (i.e. a loop) the variable is only available inside of that loop
    const
    • const is a variable that once it has been created the value can never change
    • const has a block scope
