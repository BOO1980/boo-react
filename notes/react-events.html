React Events

  • Just like HTML, React can perform actions based on user events
  • React has the same events as HTML: Click, change, mouseover

Adding Events

  • React events are written in camelCase syntax
  • React event handlers are written inside curly braces

Event Handlers

  • A good practice is to put the event handler as a method in the component class:

Bind this

  • For methods in React, the this keyword should represent the component that owns the method.
  • That is why you should use arrow functions. With arrow function, this will always represent the object that define the arrow function

Why Arrow Functions?

  • In class components, the this keyword is not defined by default, so with regular functions the this keyword represents the object that called the method, which can be the global window object, a HTML button, or whatever
  • If you must use regular functions instead of arrow functions you have to bind this to the component instance using the bind() methidcd

Passing Arguments

  • If you want to send parameters into an event handler you have two options:
        • Make and anonymous arrow function
        Bind the event handler to this
    • Note the first argument has to be this
    • If you send arguments without using the bind method, (this.shoot(this,"Goal") instead of this.shoot.bind(this,"Goal")), the shoot function will be executed when the page is loaded instead of waiting for the button to be clicked

React Event Object

  • Event hanlders have access to the React event that triggered the function
  • With the arrow function you have to send the event argument manually
  • Without arrow function, the React event object is sent automatically as the last argument with the bind() methid
